## 0. 용어 30초 정리

| 말 | 쉽게 말하면 |
|----|------------|
| **Git** | 내 컴퓨터에서 파일 변경을 기록하는 프로그램 |
| **GitHub** | 그 기록을 인터넷에 올려 같이 보는 사이트 |
| **저장소(Repository, 레포)** | 프로젝트 폴더 하나 |
| **Fork(포크)** | 원본 레포를 **내 GitHub 계정으로 복사**해 오는 것 |
| **Clone(클론)** | GitHub의 레포를 **내 컴퓨터로 내려받는** 것 |
| **Commit(커밋)** | 변경 내용을 **저장(기록)** 하는 것 |
| **Push(푸시)** | 내 커밋을 **GitHub에 올리는** 것 |
| **Pull(풀)** | GitHub의 최신 내용을 **내려받는** 것 |
| **Branch(브랜치)** | 남 작업과 안 섞이게 만드는 **내 작업 공간** |
| **PR(Pull Request)** | "내 작업 합쳐주세요" 하고 요청하는 것 |

## 1. 준비물 설치 (최초 1번)

1. **GitHub 계정** 만들기 → https://github.com (이메일로 가입)
2. **Git 설치**
   - Mac: 터미널 열고 `git --version` 입력 → 없으면 설치하라고 뜸, 시키는 대로
   - Windows: https://git-scm.com 에서 다운로드 → 계속 "다음"
3. **Git LFS 설치** (이미지·사운드 같은 큰 파일용, 꼭!)
   - Mac: `brew install git-lfs`
   - Windows: https://git-lfs.com 에서 설치
   - 설치 후 한 번: `git lfs install`
4. **Unity Hub + Unity** 설치 (버전은 팀에서 정한 것과 **똑같이**)

> 💡 **터미널**: Mac은 "터미널" 앱, Windows는 "Git Bash". 명령어를 한 줄씩 복사 → 붙여넣기 → Enter.

## 3. 내 컴퓨터로 가져오기 (클론)

터미널을 열고, 프로젝트를 둘 폴더로 이동한 뒤(예: 바탕화면):

```bash

```

그리고 **원본 저장소**도 연결해 둡니다(나중에 최신 받아오려고):

```bash

```

> `<내아이디>` 는 본인 GitHub 아이디로 바꾸세요.

## 4. ⭐ 로그인(인증) — 여기서 다들 막혀요

`git push` 할 때 GitHub가 로그인을 물어봐요. **비밀번호는 안 통해요.** 둘 중 하나로 하세요.

### 방법 A) GitHub CLI (제일 쉬움, 추천)
```bash
brew install gh        # (Windows는 https://cli.github.com)
gh auth login
```
질문이 나오면: **GitHub.com → HTTPS → Yes → Login with a web browser** 선택
→ 화면의 코드 입력하고 브라우저에서 로그인하면 끝. 이후 자동으로 됩니다.

### 방법 B) 토큰(PAT)
1. https://github.com/settings/tokens → **Generate new token (classic)**
2. `repo` 체크 → 생성 → **토큰 복사** (한 번만 보임!)
3. `git push` 할 때 비밀번호 자리에 **그 토큰을 붙여넣기**

> 🛑 **VS Code 터미널에서 이상한 에러**(`askpass.sh ... No such file`)가 나면:
> 맥 기본 **"터미널" 앱**을 따로 열어서 거기서 하면 돼요. 또는 먼저 `unset GIT_ASKPASS` 입력 후 다시 시도.

## 6. 🔁 작업하는 순서 (매번 이대로!)

### ① 작업 시작 전 — 최신 내용 받기
```bash
git checkout main
git pull upstream main
```

### ② 내 작업 브랜치 만들기 (남 작업과 안 섞이게)
```bash
git checkout -b feature/내작업이름
# 예) git checkout -b feature/library-ui
```

### ③ Unity에서 작업 (코딩/씬 편집 등)

### ④ 저장(커밋)하고 올리기(푸시)
```bash
git add .
git commit -m "feat: 무엇을 했는지 한 줄"
git push origin feature/내작업이름
```

### ⑤ GitHub에서 합치기 요청 (PR)
1. 내 포크 페이지에 가면 **`Compare & pull request`** 버튼이 떠요 → 클릭
2. 제목/내용 적고 **`Create pull request`**
3. 팀 리뷰 후 병합됨 → 끝!

> 커밋 메시지 예: `feat: 플레이어 점프 추가`, `fix: 버튼 안 눌리는 버그 수정`
> (자세한 규칙: [컨벤션.md](컨벤션.md))

## 8. 자주 나는 에러 & 해결

| 증상 | 해결 |
|------|------|
| push할 때 `Username/Password` 물어보고 안 됨 | 4번 인증(gh 또는 토큰) |
| `askpass.sh ... No such file` | 4번의 🛑 박스 참고 (기본 터미널에서 하기) |
| 이미지가 깨진 텍스트로 보임 | `git lfs install` 안 함 → 설치 후 다시 클론/pull |
| Unity 열 때 "Safe Mode?" | 컴파일 에러일 수 있음. `Ignore` 누르고 콘솔 에러 캡쳐해서 팀에 공유 |
| `한글이 □ 네모로` 보임 | Unity 메뉴 `서울플레이업 > 한글 폰트 적용` 실행 |
| 충돌(conflict) 떴어요 | 혼자 풀지 말고 **즉시 팀에 공유** |
