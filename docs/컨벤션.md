# 개발 컨벤션

협업 시 코드와 작업 방식을 통일하기 위한 규칙입니다.

## 🌿 브랜치

| 브랜치 | 용도 |
|--------|------|
| `main` | 항상 실행되는 안정 버전 (직접 push 금지) |
| `feature/기능이름` | 기능 개발 (예: `feature/player-move`) |
| `fix/버그이름` | 버그 수정 (예: `fix/enemy-collision`) |

- 작업은 항상 `feature/` 또는 `fix/` 브랜치에서 → **Pull Request**로 합칩니다.
- 원본(`mingyo07`)에는 직접 push하지 않고 PR로만 반영합니다.

## ✍️ 커밋 메시지

```
<타입>: <한 줄 요약>
```

| 타입 | 의미 |
|------|------|
| `feat` | 새 기능 |
| `fix` | 버그 수정 |
| `refactor` | 리팩토링 (기능 변화 없음) |
| `art` | 그래픽/에셋 추가·수정 |
| `docs` | 문서 |
| `chore` | 설정, 빌드 등 기타 |

예시:
```
feat: 플레이어 점프 구현
fix: 적이 벽을 통과하는 버그 수정
art: 타이틀 화면 배경 추가
```

> ❌ `기본설정`, `수정` 처럼 두루뭉술한 메시지는 피하기

## 💻 C# 코딩 규칙

| 대상 | 규칙 | 예시 |
|------|------|------|
| 클래스 / 메서드 | PascalCase | `PlayerController`, `Jump()` |
| 변수 / 매개변수 | camelCase | `moveSpeed`, `targetPos` |
| private 필드 | `_camelCase` 또는 `[SerializeField] private` | `_health` |
| 상수 | 대문자 SNAKE | `MAX_HP` |
| 한 파일 = 한 클래스 | 파일명 = 클래스명 | `PlayerController.cs` |

## 📂 에셋 규칙

- 스크립트는 **자기 게임 폴더 안** `Games/<게임명>/Scripts/` 에 둡니다 (여러 게임 공유는 `Games/_공통/Scripts/`)
- 파일/폴더 이름에 **한글·공백 지양** (영문 + 언더스코어 권장)
- `.meta` 파일은 항상 함께 커밋

## 🤝 협업 충돌 예방

- **씬·프리팹은 동시 작업 금지.** 누가 어떤 씬을 만질지 미리 정합니다 → [작업분담.md](작업분담.md)
- 작업 시작 전 `git pull upstream main`으로 최신화
- 큰 작업은 잘게 나눠 자주 커밋·PR
