## 1. 프로젝트 전체 개요

- **이름**: 서울플레이업 (Seoul Play Up)
- **무엇**: 서울의 유명 장소들을 미니게임으로 홍보하는 **Unity 게임 모음**
- **대회**: 제1회 **서울 플레이업 AI 게임 챌린지**
  - 1차 마감: **2026-06-29 23:59**
  - 규칙: AI로 에셋을 생성해야 함. **VARCO**(NCSOFT의 생성형 AI)로 이미지/3D를 만들고 **사용 증빙 필요**.
- **협업**: GitHub 포크 방식. `SeHo07/seoulplayer` (mingyo07/seoulplayer 에서 포크). 작업은 **`main` 브랜치에 직접**.
- **총 11개 장소** 게임 예정: 예술의전당, **별마당도서관**, 남산, 63타워, 북한산, 경복궁, 올림픽공원, 서울어린이대공원, 반포한강공원, 국립중앙박물관, 홍대.

### 기술 스택
- **Unity 6000.4.5f1 (Unity 6.4)**
- **URP** (Universal Render Pipeline)
- **새 Input System** (`Keyboard.current` 사용. 구 Input.GetKey 아님)
- **TextMeshPro** (한글 폰트)
- **Git LFS**: 이미지/오디오/3D/폰트 (`.png .jpg .fbx .wav .mp3 .ttf` 등)
- **에디터 자동 빌더 패턴**: 메뉴를 누르면 Editor 스크립트가 씬+오브젝트를 코드로 자동 생성 (직접 손으로 씬 조립 안 해도 됨)

### ⚠️ 과거에 크게 데인 함정 (반드시 기억)
1. **한글 폴더명 금지**: macOS와 git의 유니코드 정규화(NFC/NFD) 차이 때문에 한글 폴더명을 쓰면 **다른 팀원이 클론하면 "폴더가 비어있음"** 현상 발생. → 그래서 모든 게임 폴더를 **영어**로 바꿨음. (예: 별마당도서관 → `Library`)
2. **.gitignore의 `[Ll]ibrary/`**: Unity 캐시 폴더 `Library/`를 무시하는 규칙이 게임폴더 `Games/Library/`까지 무시해버린 버그가 있었음. → 지금은 `/gameproject/[Ll]ibrary/` 로 경로를 박아서 캐시만 무시.
3. **`MaterialLocation.External`은 Unity 6에서 obsolete** → 컴파일/임포트 막힘. `InPrefab` 써야 함.
4. 코드/이미지가 안 올라가면 **LFS 포인터 문제**일 수 있음(`git lfs pull`).

## 3. 별마당 도서관 게임 — 기획/요구사항 (사용자가 요청한 전부)

### 3.1 핵심 컨셉 (2.5D)
- **3D로 만든 별마당 도서관을 45도 위에서 내려다보는 뷰**로, **WASD로 걸어다니며 구경**한다(서울 명소 홍보 목적).
- 도서관은 **실제 별마당처럼 아주 큰** 건축물 → **플레이어보다 훨씬 커야 함**.
- 도서관(책장)에 다가가 **F**를 누르면 → **2D 책찾기 화면**으로 전환.

### 3.2 책찾기 미니게임 (2D)
- 화면에 **책장 이미지(bookshelf.png)** 가 뜨고, 우상단에 **"찾을 책"** 이 크롭되어 표시됨.
- 플레이어는 책장에서 그 책을 **클릭**으로 찾는다. 책 42권 각각이 **클릭 가능한 영역(버튼/히트영역)**.
- **점수제 = 제한시간 내에 최대한 많이 찾기**:
  - 정답 클릭 → 점수 +1, **목표 책이 즉시 새 책으로 변경**(계속 바뀜).
  - 오답 클릭 → **시간 차감**(패널티).
  - 제한시간(기본 30초) 종료 → 결과 화면에 "N권 찾았어요!".
- 비슷한 책이 많아 **적당히 어렵게**.

### 3.3 사용자가 명시한 세부 요구
- 상호작용 키 = **F** (원래 E였는데 변경 요청).
- 도서관 재질/색감이 **VARCO 사이트 렌더처럼 정확하고 밝게** 나와야 함 (Unity에서 어둡게 나오면 안 됨). ※ 사용자가 "VARCO에선 정확히 렌더된다"고 확인함 → 텍스처 자체는 멀쩡, Unity 렌더링/재질 설정 문제.
- 책장 이미지가 **반드시 떠야** 함(여러 번 안 떠서 고생 중).
- 도서관 **앞뒤가 뒤집혀** 보이면 안 됨(방향 보정 필요).
- FBX 다운로드 포맷은 **FBX가 맞음**(바꾸지 말 것).

## 4. 현재 구현 상태 (LibraryExploreBuilder.cs가 자동 생성)

메뉴 **`별마당도서관 > 플레이 가능한 씬 생성`** 을 누르면 `LibraryExplore.unity` 씬을 통째로 코드로 만든다. 생성 내용:
1. **임포트 설정**: bookshelf.png → Default 텍스처. FBX → addCollider + materialImportMode=ImportStandard + materialLocation=**InPrefab**.
2. **조명/바닥**(Plane 100유닛).
3. **InteractZone**(트리거 박스) 먼저 무조건 생성 → FBX가 실패해도 F는 작동.
4. **FBX 배치**(try/catch로 감쌈):
   - 인스턴스화 → **Y 180° 회전**(앞뒤 보정) → `FixToURP()` 재질변환.
   - **렌더러 바운드를 측정해 최대치수 40유닛으로 자동 스케일**, 바닥(y=0)에 앉히고 z≈18에 배치.
   - InteractZone 크기를 모델 바운드에 맞춤.
5. **플레이어**: (0,0,-12)에서 시작. CharacterController + Body캡슐(콜라이더 제거) + LibraryExplorer + LibraryInteract.
6. **카메라**: IsoCameraFollow, offset (0,24,-24), pitch 45.
7. **Canvas + EventSystem(InputSystemUIInputModule)** + "[ F ] 책 찾기" 프롬프트.
8. **FindScreen 패널**: ShelfImage(RawImage, texture=bookshelf) + Buttons(겹치는 영역) + TargetThumb(찾을 책 크롭) + Info텍스트 + ResultPanel. FindBookScreen·LibraryInteract 전부 코드로 연결.

### `FixToURP()` (재질 변환 — 밝기 문제의 핵심)
- Unity 6 URP는 FBX를 임포트할 때 재질을 **이미 URP/Lit**으로 만든다.
- 예전 코드는 "이미 Universal 재질이면 건너뛰기" → URP/Lit 그대로(조명영향 받아 어두움) 유지되는 버그가 있었음.
- **수정**: "이미 **Unlit**이면 건너뛰기"로 바꿔서, URP/Lit를 **URP/Unlit**으로 변환. `_BaseColor`=흰색 고정, `_BaseMap`=텍스처 → **풀 밝기**로 텍스처가 그대로 보이게.
- 이유: VARCO 텍스처(diffuse.png)는 명암이 이미 구워져 있어 Unlit이 원본 렌더와 가장 비슷.

### 주요 파라미터(인스펙터에서 조정 가능)
- `BookWall` Transform: Scale(자동 40), Rotation Y(180), Position.
- `Main Camera > IsoCameraFollow`: Offset, Pitch.
- `Player > LibraryExplorer`: MoveSpeed(8).
- `FindScreen > FindBookScreen`: Cabinets(좌/우 책장 uv 영역, 각 7열3행), TimeLimit(30), WrongPenalty(1.5).

## 5. 현재 진행 중인 디버깅 / 미해결 가능성

지금까지 **블라인드(직접 실행 못 보고)로 고쳐와서** 여러 번 빗나갔다. 최근 커밋들로 아래를 고쳤지만 **사용자 Unity에서 실제 검증이 아직 필요**:

| 증상 | 추정 원인 | 적용한 수정 |
|---|---|---|
| 색감 어두움 | URP/Lit이 Unlit로 변환 안 됨(건너뛰기 조건) | 조건 Universal→Unlit 으로 수정 |
| F 눌러도 책찾기 화면 안 뜸 | 빌드가 FBX 처리 중 예외로 **중간에 멈춰** 책찾기 UI가 안 만들어짐 | try/catch + InteractZone 무조건 생성 |
| 도서관 작음 | FBX 원본 스케일 작음 | 바운드 측정해 40유닛 자동 스케일 |
| 도서관 앞뒤 뒤집힘 | 모델 정면이 카메라 반대 | Y 180° 회전 |
| 콘솔 에러 `WindowLayout.FindRootSplitView` | **Unity 에디터 자체 버그**(내 코드 아님). 빌드 중 터지면 씬생성 중단 가능 | 사용자에게 Window>Layouts>Default로 리셋 안내 |

### 다음 작업자가 확인할 것
1. 메뉴로 씬 재생성 후 **콘솔에 `[LibraryExploreBuilder]` 에러가 있는지** (있으면 그게 진짜 원인).
2. Play → WASD → F → **책찾기 화면(bookshelf.png)이 뜨는지**.
3. 도서관 **색감이 밝아졌는지**, **방향이 맞는지**.
4. 책 42칸 클릭 영역이 이미지와 맞는지(`FindBookScreen.Cabinets` uv 미세조정 필요할 수 있음).

### 검증 팁
- 사용자는 한국어로 소통하며, 코딩 초보에 가깝다 → **터미널 명령/Unity 메뉴 클릭을 구체적으로** 안내할 것.
- bookshelf.png, FBX는 이 사용자 컴퓨터에선 **실제 파일 확인됨**(LFS 포인터 아님).
- 사용자가 "직접 실행해가며 단계별로 확인하라"고 요청 → 가능하면 추측 대신 **콘솔 에러/스크린샷 기반**으로 진단할 것. (화면제어 computer-use도 가능)
